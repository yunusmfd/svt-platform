<a name="1"></a>
## 1. نظرة عامة على المنصة

| الخاصية | الوصف |
|---------|-------|
| **النوع** | موقع ساكن (Static) — بلا خادم وبلا قاعدة بيانات |
| **التقنيات** | HTML5 + CSS3 + JavaScript عادي (وحدات ES) + بيانات JSON |
| **اللغات** | العربية والفرنسية (تبديل بنقرة) |
| **المظهر** | فاتح / ليلي / حسب النظام |
| **يعمل دون إنترنت** | نعم (PWA) بعد أول زيارة، وقابل للتثبيت كتطبيق |
| **الاستضافة** | GitHub Pages أو Netlify — مجاناً ودون إعداد |

**بنية المشروع باختصار:**

```
nova-svt/
├── index.html                  ← نقطة الدخول (لا تحتاج تعديلها غالباً)
├── manifest.webmanifest        ← إعداد التطبيق (PWA)
├── sw.js                       ← العمل دون إنترنت
│
├── assets/
│   ├── css/                    ← التنسيقات (الألوان، الخطوط، التصميم)
│   ├── js/                     ← الكود (نادراً ما تلمسه)
│   └── icons/                  ← الشعار والأيقونات
│
├── data/                       ⭐ هنا يعيش المحتوى الذي تتحكّم فيه
│   ├── levels.json             ← المستويات الدراسية ووحدات منهاج كل مستوى
│   ├── blog.json               ← منشورات المدونة (أخبار، توجيهات، إعلانات)
│   ├── experiments.json        ← تجارب معرض المختبر (فيديو/animation/تفاعلية)
│   └── lessons/                ← درس لكل مستوى
│       ├── 1ac.json
│       ├── 2ac.json
│       ├── 3ac.json
│       ├── tcs.json
│       ├── 1bac.json
│       └── 2bac.json
│
└── docs/                       ← الأدلّة
```

👈 **٩٠٪ من عملك سيكون داخل مجلّد `data/` فقط.**

<a name="2"></a>
## 2. تشغيل المنصة محلياً (قبل النشر)

> ⚠️ **مهم جداً:** لا تفتح `index.html` بالنقر المزدوج مباشرةً! ستظهر صفحة بيضاء.
> السبب: المنصة تستعمل وحدات JavaScript وتحميل ملفات JSON، وهذه تتطلّب **خادماً محلياً** ولا تعمل عبر بروتوكول `file://`.

اختر إحدى الطرق:

### الطريقة 1: VS Code + Live Server (الأسهل والمنصوح بها)
1. ثبّت محرّر **VS Code** (مجاني).
2. ثبّت إضافة **Live Server** من سوق الإضافات.
3. افتح مجلّد المشروع في VS Code.
4. انقر بزرّ الفأرة الأيمن على `index.html` → **Open with Live Server**.
5. سيُفتح المتصفّح على `http://127.0.0.1:5500` — والمنصة تعمل.

### الطريقة 2: Python (إن كان مثبّتاً)
افتح Terminal داخل مجلّد المشروع ونفّذ:
```bash
python3 -m http.server 8000
```
ثم افتح: `http://localhost:8000`

### الطريقة 3: Node.js
```bash
npx serve
```

✅ مع أي طريقة من هذه، تعمل كل المزايا (تبديل اللغة، الوضع الليلي، الاختبارات، **والعمل دون إنترنت** لأن `localhost` يُعتبر اتصالاً آمناً).

<a name="3"></a>
## 3. النشر على GitHub Pages

نشر مجاني ودائم. اتبع الخطوات بالترتيب:

### الخطوة 1: إنشاء حساب ومستودع
1. أنشئ حساباً على [github.com](https://github.com) (إن لم يكن لديك).
2. انقر **+** (أعلى يمين) → **New repository**.
3. اسم المستودع: مثلاً `nova-svt`. اجعله **Public**. لا تُضف README. → **Create repository**.

### الخطوة 2: رفع الملفات
أسهل طريقة (عبر المتصفّح):
1. في صفحة المستودع الجديد، انقر **uploading an existing file**.
2. ⚠️ **افتح مجلّد `nova-svt` وحدّد كل ما بداخله** (الملفات والمجلّدات)، ثم اسحبها — **وليس المجلّد نفسه**. يجب أن يكون `index.html` في **جذر المستودع** مباشرةً.
3. انقر **Commit changes**.

### الخطوة 3: تفعيل Pages
1. في المستودع: **Settings** → **Pages** (القائمة الجانبية).
2. تحت **Source**: اختر **Deploy from a branch**.
3. **Branch**: `main` — والمجلّد `/ (root)` → **Save**.
4. انتظر دقيقة، ثم حدّث الصفحة — سيظهر رابط موقعك:
   ```
   https://اسم-حسابك.github.io/nova-svt/
   ```

✅ **يعمل مباشرةً دون أي إعداد**، لأن المنصة تستعمل مسارات نسبية تتوافق مع المجلّد الفرعي.

### التحديث لاحقاً
- لتعديل أي ملف: افتحه في GitHub → أيقونة القلم ✏️ → عدّل → **Commit**. أو ارفع نسخة جديدة.
- يُعاد النشر تلقائياً خلال ~دقيقة.
- 🔴 **بعد أي تعديل، ارفع رقم النسخة في `sw.js`** (انظر [القسم 12](#12)) كي يرى الزوّار التحديث فوراً.

<a name="4"></a>
## 4. النشر على Netlify

أسرع، ويمنحك HTTPS تلقائياً ورابطاً أجمل.

### الطريقة 1: السحب والإفلات (فورية)
1. أنشئ حساباً على [netlify.com](https://netlify.com).
2. في لوحة التحكّم: **Add new site** → **Deploy manually**.
3. **اسحب مجلّد `nova-svt` كاملاً** إلى المنطقة المخصّصة.
4. يُنشر فوراً على رابط عشوائي مثل `random-name.netlify.app`.
5. لتغيير الاسم: **Site configuration** → **Change site name**.

### الطريقة 2: مرتبط بـ GitHub (الأفضل للاستمرار)
1. **Add new site** → **Import an existing project** → **GitHub**.
2. اختر مستودع `nova-svt`.
3. الإعدادات:
   - **Build command**: اتركه **فارغاً**.
   - **Publish directory**: `/` (إن كان `index.html` في الجذر).
4. **Deploy**.

✅ بهذه الطريقة، **كل تعديل ترفعه على GitHub يُنشر تلقائياً على Netlify**.

**مزايا Netlify:** HTTPS تلقائي (ضروري لميزة العمل دون إنترنت)، نشر تلقائي، إمكانية ربط نطاق خاص (domain) لاحقاً.

<a name="5"></a>
## 5. إضافة درس جديد ⭐

هذا أكثر ما ستفعله. **لا كود — JSON فقط.**

### أين؟
افتح ملف المستوى المناسب داخل `data/lessons/`:

| الملف | المستوى |
|-------|---------|
| `1ac.json` | الأولى إعدادي |
| `2ac.json` | الثانية إعدادي |
| `3ac.json` | الثالثة إعدادي |
| `tcs.json` | الجذع المشترك العلمي |
| `1bac.json` | الأولى باك علوم |
| `2bac.json` | الثانية باك علوم |

### كيف؟
الملف عبارة عن **قائمة `[ ... ]`** تحوي دروساً (كل درس كائن `{ ... }`). أضِف كائن درسك الجديد، **مفصولاً بفاصلة `,`** عن الدرس السابق.

### شرح حقول الدرس

| الحقل | إلزامي | الوصف | القيم/المثال |
|-------|:------:|-------|--------------|
| `id` | ✅ | معرّف فريد للدرس (في الرابط) | أحرف لاتينية صغيرة وشرطات: `"photosynthese"` |
| `level` | ✅ | رمز المستوى | `"1ac"`, `"2bac"`... |
| `branch` | ✅ | الفرع | `"bio"` (حياة) أو `"geo"` (أرض) |
| `fig` | ✅ | خطاطة صورة البطاقة | `cell`/`dna`/`tecto`/`gas`/`digest` (انظر [القسم 7](#7)) |
| `duration` | ✅ | مدّة التعلّم بالدقائق | `45` |
| `unit` | ✅ | اسم الوحدة | `{ "ar": "...", "fr": "..." }` |
| `title` | ✅ | عنوان الدرس | `{ "ar": "...", "fr": "..." }` |
| `desc` | ✅ | وصف مختصر (يظهر في البطاقة) | `{ "ar": "...", "fr": "..." }` |
| `video` | ✅ | عنوان الفيديو ومدّته | `{ "title": {ar,fr}, "length": "07:20" }` |
| `intro` | ✅ | فقرات المقدّمة (قائمة) | فقرة أو أكثر، يُسمح فيها بوسوم HTML بسيطة مثل `<strong>` |
| `figs` | ✅ | الخطاطات داخل الدرس | قائمة `{ key, label{ar,fr}, cap{ar,fr} }` |
| `quiz` | ✅ | أسئلة الاختبار | قائمة أسئلة (انظر أدناه) |

**حقل السؤال في الاختبار:**

| الحقل | الوصف |
|-------|-------|
| `q` | نصّ السؤال `{ar, fr}` |
| `options` | قائمة من **4 خيارات** `{ar, fr}` |
| `correct` | **رقم الخيار الصحيح، يبدأ من 0** (0 = الأول، 1 = الثاني...) |
| `explain` | شرح يظهر بعد الإجابة `{ar, fr}` |

### 📋 قالب جاهز للنسخ

انسخ هذا، الصقه داخل ملف المستوى (وأضف فاصلة قبله إن وُجد درس سابق)، ثم عدّل القيم:

```json
{
  "id": "photosynthese",
  "level": "tcs",
  "branch": "bio",
  "fig": "cell",
  "duration": 50,
  "unit": {
    "ar": "الوحدة 3: التغذية عند النباتات الخضراء",
    "fr": "Unité 3 : La nutrition chez les plantes vertes"
  },
  "title": {
    "ar": "التركيب الضوئي",
    "fr": "La photosynthèse"
  },
  "desc": {
    "ar": "كيف تصنع النباتات الخضراء مادتها العضوية انطلاقاً من الماء وثنائي أكسيد الكربون والضوء.",
    "fr": "Comment les plantes vertes fabriquent leur matière organique à partir d'eau, de CO₂ et de lumière."
  },
  "video": {
    "title": { "ar": "آلية التركيب الضوئي", "fr": "Mécanisme de la photosynthèse" },
    "length": "08:30"
  },
  "intro": [
    {
      "ar": "يُعدّ <strong>التركيب الضوئي</strong> العملية التي تصنع بها النباتات الخضراء المادة العضوية اعتماداً على طاقة الضوء.",
      "fr": "La <strong>photosynthèse</strong> est le processus par lequel les plantes vertes fabriquent la matière organique grâce à l'énergie lumineuse."
    }
  ],
  "figs": [
    {
      "key": "cell",
      "label": { "ar": "الشكل 1 — الصانعة الخضراء", "fr": "Fig. 1 — Le chloroplaste" },
      "cap": { "ar": "مقر التركيب الضوئي", "fr": "Siège de la photosynthèse" }
    }
  ],
  "quiz": [
    {
      "q": { "ar": "أين يحدث التركيب الضوئي؟", "fr": "Où a lieu la photosynthèse ?" },
      "options": [
        { "ar": "في النواة", "fr": "Dans le noyau" },
        { "ar": "في الصانعات الخضراء", "fr": "Dans les chloroplastes" },
        { "ar": "في الميتوكندري", "fr": "Dans les mitochondries" },
        { "ar": "في الجدار", "fr": "Dans la paroi" }
      ],
      "correct": 1,
      "explain": {
        "ar": "تحتوي الصانعات الخضراء على اليخضور الذي يلتقط الضوء.",
        "fr": "Les chloroplastes contiennent la chlorophylle qui capte la lumière."
      }
    }
  ]
}
```

### ✅ قواعد ذهبية
- **كل نصّ يجب أن يكون بالعربية والفرنسية** (`ar` و`fr`).
- **`id` فريد** ولا يتكرّر، بأحرف لاتينية صغيرة وشرطات فقط.
- **لا تضع فاصلة بعد آخر عنصر** في أي قائمة أو كائن (خطأ شائع جداً).
- بعد التعديل، **تحقّق من صحة JSON** عبر موقع مثل [jsonlint.com](https://jsonlint.com).

<a name="6"></a>
## 6. إضافة مستوى دراسي جديد

خطوتان فقط:

### الخطوة 1: سجّل المستوى في `data/levels.json`
أضف سطراً جديداً (مع فاصلة):
```json
{ "id": "2bac-svt", "stage": "lycee", "code": "2BAC", "name": { "ar": "الثانية باك علوم تجريبية", "fr": "2ème Bac SVT" } }
```
| الحقل | القيم |
|-------|-------|
| `id` | معرّف فريد (سيكون اسم ملف الدروس) |
| `stage` | `"college"` (إعدادي) أو `"lycee"` (ثانوي) — يحدّد التجميع في صفحة الدروس |
| `code` | رمز قصير يظهر على الزرّ مثل `"2BAC"` |
| `name` | الاسم الكامل `{ar, fr}` |
| `units` | قائمة وحدات المنهاج الرسمية لهذا المستوى (اختياري لكن مستحسن) — انظر أدناه |

**حقل `units` (وحدات المنهاج):** تعرضه بوّابة الدروس كسلّم رسمي لوحدات المستوى، وتُستعمل نصوصه لتجميع بطاقات الدروس تحت عنوان وحدتها (عبر تطابق حقل `unit` في كل درس مع أحد عناصر هذه القائمة):
```json
"units": [
  { "num": 1, "ar": "الوحدة 1: ...", "fr": "Unité 1 : ..." },
  { "num": 2, "ar": "الوحدة 2: ...", "fr": "Unité 2 : ..." }
]
```
عند إضافة درس لهذا المستوى، اجعل حقل `unit` في الدرس (انظر [القسم 5](#5)) مطابقاً **حرفياً** لأحد عناصر `units` هنا، ليظهر الدرس مجمَّعاً تحت وحدته الصحيحة.

### الخطوة 2: أنشئ ملف الدروس
أنشئ ملفاً جديداً `data/lessons/2bac-svt.json` (نفس `id`)، يبدأ بقائمة فارغة أو بدرس:
```json
[]
```
ثم أضف الدروس كما في [القسم 5](#5).

✅ يظهر المستوى تلقائياً في صفحة الدروس — **دون لمس أي كود**.

<a name="7"></a>
## 7. الخطاطات العلمية (الرسوم)

المنصة تتضمّن رسوماً علمية متجهيّة (SVG) جاهزة، تُستعمل في صورة بطاقة الدرس (`fig`) وداخل الدرس (`figs`).

| المفتاح | الرسم | مناسب لـ |
|---------|-------|----------|
| `cell` | الخلية | البيولوجيا العامة، التركيب الضوئي |
| `dna` | الحمض النووي ADN | الوراثة، الجينات |
| `tecto` | الصفائح التكتونية | الجيولوجيا، الزلازل |
| `gas` | المبادلات الغازية | التنفس |
| `digest` | الجهاز الهضمي | الهضم، التغذية |

استعملها هكذا داخل الدرس:
```json
"fig": "dna",
"figs": [
  { "key": "tecto", "label": {...}, "cap": {...} }
]
```

### لإضافة خطاطة جديدة
يتطلّب هذا **تعديلاً بسيطاً في الكود** (لأن الرسوم مدمجة كـ SVG). افتح `assets/js/core/icons.js`، وابحث عن `export const FIGURES = {`، وأضف مفتاحاً جديداً بكود SVG على نمط الموجود. إن لم تكن مرتاحاً لذلك، استعمل أقرب خطاطة متاحة، أو اطلب المساعدة في إضافتها.

<a name="8"></a>
## 8. الملفات القابلة للتحميل

في صفحة كل درس، قسم **"الملفات"** يعرض حالياً ثلاثة عناصر تُولَّد تلقائياً من بيانات الدرس:
- **ملخّص الدرس** (نصّي).
- **الخطاطة العلمية** (صورة PNG).
- **سلسلة تمارين** (PDF — معلَّمة "قريباً").

> 🔎 **بصراحة:** هذه العناصر حالياً مولّدة تلقائياً، وزرّ التمارين تجريبي. **إرفاق ملفات PDF حقيقية (سلاسل تمارين، فروض)** ميزة تتطلّب تطويراً بسيطاً (إضافة حقل `files` للدرس ووصله بملفات فعلية في مجلّد `assets/`). إن أردت تفعيلها فعلياً، أخبرني وأضيفها لك — وهي إضافة مفيدة جداً لأستاذ.

<a name="9"></a>
## 9. إضافة صفحة أو قسم جديد

هناك حالتان:

### الحالة (أ): إضافة قسم داخل صفحة موجودة — *الأسهل*
مثال: إضافة قسم "آراء التلاميذ" في الصفحة الرئيسية.
1. افتح ملف العرض المناسب في `assets/js/views/` (مثلاً `home.js`).
2. أضف كتلة HTML جديدة داخل القالب.
3. ضع نصوصها في `assets/js/core/i18n.js` (انظر [القسم 10](#10)) واستدعها بـ `${ui("مفتاحك")}`.

### الحالة (ب): إضافة صفحة جديدة كاملة — *6 خطوات*
مثال: صفحة "اتصل بنا" على الرابط `#/contact`.

**1) سجّل المسار** في `assets/js/core/config.js` داخل `ROUTES`:
```js
export const ROUTES = {
  home: "home",
  lessons: "lessons",
  detail: "detail",
  lab: "lab",
  about: "about",
  contact: "contact",   // ← جديد
};
```

**2) أنشئ ملف العرض** `assets/js/views/contact.js`:
```js
import { app } from "../core/dom.js";
import { ui } from "../core/i18n.js";

export function renderContact() {
  app().innerHTML = `
    <section class="section">
      <div class="wrap">
        <h1 class="section-title">${ui("contact_title")}</h1>
        <p class="section-head">${ui("contact_desc")}</p>
      </div>
    </section>
  `;
}
```

**3) أضف المسار** في `assets/js/core/router.js` داخل دالة `parseHash` (في كتلة `switch`):
```js
case "contact":
  return { view: ROUTES.contact };
```

**4) اربط العرض** في `assets/js/main.js`:
```js
// أعلى الملف مع بقية الاستيرادات:
import { renderContact } from "./views/contact.js";

// داخل switch في دالة render:
case ROUTES.contact:
  renderContact();
  break;
```

**5) أضف رابط التنقّل** في `index.html` — **مرّتين** (في `<nav class="nav-links">` وفي القائمة الجانبية `#drawer`):
```html
<a class="nav-link" data-route="contact" href="#/contact" data-i18n="nav_contact"></a>
```
(في القائمة الجانبية أضف أيضاً `data-drawer-close` في نهاية الوسم.)

**6) أضف النصوص** في `assets/js/core/i18n.js` (انظر [القسم 10](#10)):
```js
nav_contact: { ar: "اتصل بنا", fr: "Contact" },
contact_title: { ar: "اتصل بنا", fr: "Contactez-nous" },
contact_desc: { ar: "...", fr: "..." },
```

> 💡 (اختياري) لعنوان تبويب مخصّص للصفحة، أضف حالتها في دالة `updateTitle` داخل `main.js`.

<a name="12"></a>
## 12. الصيانة وأمور مهمّة

### 🔴 الأهمّ: تحديث النسخة بعد أي تعديل
المنصة تخزّن ملفاتها للعمل دون إنترنت. بعد رفع أي تعديل، افتح `sw.js` وارفع رقم النسخة:
```js
const VERSION = "nova-svt-v1";   // غيّرها إلى "nova-svt-v2"، ثم v3...
```
بدون هذا، قد يبقى الزوّار على النسخة القديمة المخزّنة لديهم.

### التحقّق من JSON
الأخطاء الأكثر شيوعاً عند إضافة درس: **فاصلة زائدة** بعد آخر عنصر، أو **علامة اقتباس ناقصة**. تحقّق دائماً عبر [jsonlint.com](https://jsonlint.com) قبل الرفع.

### النسخ الاحتياطي
احتفظ دائماً بنسخة من مجلّد المشروع كاملاً قبل أي تعديل كبير. إن استعملت GitHub، فلديك نسخة محفوظة تلقائياً مع كل تعديل.

### ملاحظة أمنية
- إجابات الاختبارات موجودة في ملفات JSON الظاهرة للزوّار — هذا **مناسب لتقويم ذاتي تكويني**، لكن **لا تعتمده لامتحان مُقيَّم رسمياً**.
- لا تضع أبداً كلمات مرور أو أسراراً في الكود (المنصة عمومية بالكامل).

<a name="13"></a>
## 13. حلّ المشكلات الشائعة

| المشكلة | السبب والحلّ |
|---------|--------------|
| **صفحة بيضاء عند فتح `index.html` بالنقر** | `file://` لا يعمل. استعمل خادماً محلياً ([القسم 2](#2)) — أو افتح الموقع المنشور. |
| **درس جديد لا يظهر** | خطأ في JSON (فاصلة زائدة/اقتباس ناقص)، أو `id` مكرّر، أو الملف لم يُحفظ. تحقّق عبر jsonlint. |
| **التعديلات لا تظهر بعد النشر** | الذاكرة المؤقتة. ارفع `VERSION` في `sw.js`، ثم في المتصفّح اعمل Hard Refresh (`Ctrl+Shift+R`). |
| **الشعار/الأيقونة لا يتحدّثان** | يخزّنهما عامل الخدمة. ارفع `VERSION` في `sw.js`. |
| **الخطوط لا تظهر بشكلها الصحيح** | تحقّق من الاتصال بالإنترنت (خطوط Google)، أو من صحّة رابط الخطوط في `index.html`. |
| **العمل دون إنترنت لا يعمل محلياً** | يحتاج `localhost` أو HTTPS — لا يعمل عبر `file://`. على GitHub Pages وNetlify يعمل تلقائياً. |

<a name="14"></a>
## 14. إضافة منشور للمدونة

المدونة (`#/blog`) تعرض منشورات من `data/blog.json` — قائمة كائنات، كل واحد بهذا الشكل:

```json
{
  "id": "titre-unique",
  "date": "2026-09-01",
  "tag": "news",
  "title": { "ar": "عنوان المنشور", "fr": "Titre de l'article" },
  "body": [
    { "ar": "فقرة أولى...", "fr": "Premier paragraphe..." },
    { "ar": "فقرة ثانية اختيارية...", "fr": "Deuxième paragraphe (optionnel)..." }
  ]
}
```

| الحقل | الوصف |
|-------|-------|
| `id` | معرّف فريد للمنشور. |
| `date` | تاريخ بصيغة `YYYY-MM-DD` — المنشورات تُرتَّب تلقائياً من الأحدث إلى الأقدم. |
| `tag` | صنف المنشور: `"news"` (خبر)، `"guide"` (توجيه)، أو `"announcement"` (إعلان). |
| `title` | عنوان المنشور `{ar, fr}`. |
| `body` | فقرات المنشور (قائمة)، كل فقرة `{ar, fr}`. |

أضِف الكائن الجديد داخل قائمة `data/blog.json` (مع فاصلة قبله إن وُجد منشور سابق) — يظهر فوراً في `#/blog` دون أي تعديل برمجي.

<a name="15"></a>
## 15. إضافة تجربة لمعرض المختبر

معرض التجارب أسفل المختبر الافتراضي (`#/lab`) يعرض عناصر `data/experiments.json` — قائمة كائنات:

```json
{
  "id": "experience-unique",
  "type": "video",
  "title": { "ar": "عنوان التجربة", "fr": "Titre de l'expérience" },
  "desc": { "ar": "وصف مختصر.", "fr": "Brève description." },
  "src": "https://.../video.mp4"
}
```

| الحقل | الوصف |
|-------|-------|
| `id` | معرّف فريد للتجربة. |
| `type` | `"video"` (فيديو)، `"animation"` (رسم متحرك/GIF)، أو `"interactive"` (صفحة تفاعلية مُضمَّنة عبر iframe). |
| `title` | عنوان التجربة `{ar, fr}`. |
| `desc` | وصف مختصر يظهر أسفل المعاينة إن لم يُضَف الرابط بعد `{ar, fr}`. |
| `src` | رابط الملف (فيديو/صورة متحركة) أو الصفحة التفاعلية. اتركه فارغاً `""` لعرض رسالة "قريباً" بدل معاينة مكسورة. |

أضِف الكائن الجديد داخل قائمة `data/experiments.json` — تظهر التجربة فوراً كبطاقة في معرض المختبر.

## 16. خلاصة سريعة — أين أعدّل ماذا؟

| أريد أن... | الملف | كيف |
